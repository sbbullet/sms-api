The `managers` folder contains various manager classes responsible for handling different aspects of the application logic. Each manager class is designed to encapsulate specific functionalities and operations, ensuring a modular and organized codebase. By following the naming conventions, we maintain consistency and readability across the project.

-   **Folder Naming**: Use `snake_case` for folder names.
-   **File Naming**: Use `CamelCase` for file names.
-   **Naming Convention**: Do not include the word "manager" in the folder or file names.

These guidelines help in maintaining a clean and efficient project structure.
